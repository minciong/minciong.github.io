Portfolio


E-Library - Library for E-books.


This application was created to be similar to a library in which a user could borrow books in PDF form. It allowed for the study of database fundamentals. However, this program did not use SQL or and query based language to do database work. It instead used files to keep track of Users, books, and related attributes one might find in a database. The system also had a privilege system. In order to gain points to borrow more books, users had to submit more original books, which then had to be approved by a super user. A super user was similar to a user, but with extra privileges.

I was responsible for creating the Graphic User Interface, as well as creating the framework that the database was based on. I utilized Java in order to add functionality to each of the buttons, such as adding an e-book to a user's favourite books, logging down a user's read history, or submitting a new book for approval. I also set up a system in which super users could evenly split up the work of books being submitted for approval and point allocation given a multitude of submitted books.

Magical Music Player - Music Player with playlist capabilities.

Known as QTunes, this project was my introduction to QT and openGL. It was supposed to be an MP3 Player akin to iTunes in which people could create custom playlists to listen to while browsing the cover arts of the album art. My group decided to make the entire application customizable. Every widget on the application, from the playlist, to the list of total songs, to the album cover flow could be placed anywhere else, or toggled off. My main responsibility was creating the cover flow. The application added album covers to the coverflow list when songs were added to a playlist, and one could cycle through the covers using the arrow keys.

Stock Market GUI - APP to track down Stock Market Quotes.

This program was meant to simulate a stock market exchange. The application was a graphic user interface that allowed someone to search for a stock and keep track of what was bought and sold. This was originally meant to be a five person project, but three of the five were unable to do any work, so this project alos shows how much I'm capable of doing with increased pressure. I was in charge of connecting to the database, creating the queries, and making the GUI using my limited knowledge of python at the time.
